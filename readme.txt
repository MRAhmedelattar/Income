
مخطط ومكونات المشروع الكاملة
نظرة عامة على المشروع
هذا تطبيق مكتبي لإدارة الإيرادات والتحصيلات المالية، مبني باستخدام React و Electron. يدعم الوضع المظلم/الفاتح، ويحفظ البيانات محلياً باستخدام SQLite.

هيكل المشروع

revenue-system-app/
├── main.js                    # ملف الإطلاق الرئيسي لـ Electron
├── preload.js                 # جسر الاتصال بين Electron و React
├── package.json               # إعدادات المشروع والتبعيات
├── tailwind.config.js         # إعدادات Tailwind CSS
├── public/
│   └── index.html             # قالب HTML الأساسي
├── src/
│   ├── index.js               # نقطة دخول React
│   ├── index.css              # أنماط CSS الأساسية
│   ├── output.css             # ملف CSS المجمع من Tailwind
│   ├── App.jsx                # المكون الرئيسي للتطبيق
│   ├── components/            # مكونات واجهة المستخدم
│   │   ├── AppHeader.jsx      # رأس التطبيق
│   │   ├── Sidebar.jsx        # اللوحة الجانبية الرئيسية
│   │   ├── Icons.jsx          # مكتبة الأيقونات
│   │   ├── LogoPreview.jsx    # معاينة الشعارات
│   │   └── settings/          # مكونات إعدادات النظام
│   │       ├── FundsManagement.jsx    # إدارة الصناديق
│   │       ├── ItemsManagement.jsx    # إدارة بنود التحصيل
│   │       └── RevenueManagement.jsx  # إدارة الإيرادات
│   └── views/                 # صفحات التطبيق الرئيسية
│       ├── Collections.jsx    # صفحة التحصيلات
│       ├── Reports.jsx        # صفحة التقارير
│       └── SystemSetup.jsx    # صفحة إعدادات النظام
└── build/                     # ملفات البناء المجمعة
التبعيات الرئيسية
React 18: لإنشاء واجهة المستخدم
Electron: لتحويل التطبيق إلى تطبيق مكتبي
Tailwind CSS: للتصميم والأنماط
SQLite3: لقاعدة البيانات (مخطط للاستخدام المستقبلي)
jsPDF: لتصدير التقارير PDF
xlsx: لتصدير البيانات Excel
وظائف الملفات الرئيسية
1. main.js (Electron Main Process)
إنشاء النافذة الرئيسية للتطبيق
إعداد مسارات البيانات المحلية
معالجة IPC للحفظ والتحميل من الملفات
إدارة دورة حياة التطبيق
2. preload.js (Context Bridge)
يوفر واجهة آمنة بين Electron و React
يعرض دوال saveData, loadData, log للاستخدام في React
3. src/App.jsx (المكون الرئيسي)
إدارة الحالة العامة للتطبيق (الإعدادات، الصناديق، البنود، التحصيلات)
تبديل الوضع المظلم/الفاتح
تحميل وحفظ البيانات عبر Electron API
عرض المكونات الفرعية حسب الصفحة النشطة
4. src/components/Sidebar.jsx (اللوحة الجانبية)
عرض شعارات الجامعة والكلية
أزرار التنقل بين الصفحات الرئيسية
تصميم متجاوب مع الوضع RTL
5. src/views/SystemSetup.jsx (إعدادات النظام)
لوحة جانبية داخلية للتبويبات
إدارة معلومات المؤسسة، الصناديق، بنود التحصيل، والإيرادات
استيراد مكونات فرعية لكل تبويب
6. src/views/Collections.jsx (التحصيلات)
نموذج إدخال بيانات التحصيل
عرض قائمة التحصيلات السابقة
حساب الإجماليات وحفظ البيانات
7. src/views/Reports.jsx (التقارير)
عرض إحصائيات التحصيلات
تصدير التقارير (مخطط للتطوير)
8. مكونات الإعدادات (settings/)
FundsManagement: إضافة وحذف الصناديق الخاصة
ItemsManagement: إدارة بنود التحصيل مع ربطها بالصناديق
RevenueManagement: إدارة الإيرادات والمبالغ المختلفة
تدفق البيانات
البيانات تُحفظ محلياً في ملف JSON عبر Electron
React يدير الحالة المحلية ويحفظ التغييرات تلقائياً
SQLite مخطط للاستخدام المستقبلي لقاعدة بيانات أكثر تعقيداً
الميزات الرئيسية
واجهة عربية مع دعم RTL كامل
وضع مظلم/فاتح
إدارة شاملة للتحصيلات المالية
تصدير البيانات والتقارير
تصميم متجاوب وحديث
هذا المشروع يمثل نظاماً متكاملاً لإدارة الإيرادات المالية في المؤسسات التعليمية والإدارية.
